## Traefik开启监控, 日志, 追踪需要的参数

<br />

### 监控 - METRICS

METRICS 功能用于收集和展示应用程序的指标数据, 例如请求总数、请求成功率、请求响应时间、请求吞吐量等信息, 方便用户进行应用程序的监控和分析

Traefik 支持多种格式的 METRICS, 例如 Prometheus、Datadog、StatsD 等，用户可以根据需要选择适合自己的格式和工具，并进行相应的配置和启用

<br />

### 日志 - ACCESSLOG

ACCESSLOG 功能用于记录请求的详细信息, 包括请求方法、URL、响应状态码、响应时间、客户端 IP、用户代理等信息, 方便用户进行访问日志的分析和审计

Traefik 支持多种格式的 ACCESSLOG, 例如 Apache、JSON、CSV 等，用户可以根据需要选择适合自己的格式和工具，并进行相应的配置和启用

<br />

### 追踪 - TRACING
TRACING 功能用于追踪请求的流程和调用链, 方便用户进行请求的跟踪和分析

Traefik 支持多种格式的 TRACING, 例如 Jaeger、Zipkin、OpenTracing 等，用户可以根据需要选择适合自己的格式和工具，并进行相应的配置和启用

使用 TRACING 功能可以帮助用户快速定位请求的瓶颈和故障点，从而提高应用程序的性能和可靠性
